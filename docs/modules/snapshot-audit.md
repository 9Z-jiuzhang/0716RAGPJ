# 5.8 快照、回退与审计

> 来源：AI 知识库 RAG 平台产品手册（合并版）V2.1 §5.8

## 5.8.1 功能点

| 功能 | 说明 |
|------|------|
| 自动快照 | 文档上传/删除/规范化/分段规则变更/批量重分段/向量化发布/知识库权限变更前，自动创建可恢复快照 |
| 手动快照 | 管理员手动创建命名快照 |
| 快照列表 | 查看知识库的所有历史快照，按时间倒序 |
| 快照详情 | 查看快照包含的文档列表、分段数量、分段规则、权限配置等信息 |
| 差异预览 | 回退前预览将要变更的内容（哪些文档将被新增/删除/修改） |
| 选择性恢复 | 支持选择性恢复文档或恢复整个知识库 |
| 回退操作 | 将知识库恢复到指定快照状态 |
| 快照清理 | 删除不需要的旧快照，释放存储空间 |

## 5.8.2 快照内容

快照包含知识库元数据、文档版本、分段版本、索引版本、分段规则和权限配置引用。向量数据本身不直接存储在快照中，回退时通过文档元数据重建向量索引。

## 5.8.3 回退流程

```
管理员选择知识库或文档快照
  -> 查看版本差异和影响范围
  -> 二次确认
  -> 自动创建"回退前保护快照"（安全保护）
  -> 以快照内容创建新的恢复版本（不覆盖原历史）
  -> 重建受影响索引
  -> 原子切换当前版本并记录审计日志
```

**关键约束：** 恢复操作生成一条新的索引版本，不覆盖原历史。

## 5.8.4 快照策略

| 策略项 | 默认值 | 说明 |
|--------|--------|------|
| 最大快照数量 | 50 | 每个知识库最多保留的快照数，超出自动清理最早快照 |
| 自动快照触发 | 文档变更时 | 上传/删除/重新分段/重新向量化/权限变更操作前触发 |
| 快照保留天数 | 90 天 | 超过天数的快照自动清理 |
| 回退保护快照 | 始终创建 | 回退前自动创建保护快照，确保可撤销回退操作 |

## 5.8.5 审计日志

所有关键操作写入审计日志：操作者、时间、对象、前后版本、请求标识、结果与失败原因。审计日志存储于 PostgreSQL（`audit_logs` 表）。

## 本工作区实现说明

| 路径 | 职责 |
|------|------|
| `backend/app/models/snapshot.py` / `audit_log.py` / `index_version.py` | ORM |
| `backend/app/schemas/snapshot.py` / `audit.py` | 请求响应契约 |
| `backend/app/repositories/snapshot.py` / `audit.py` | 数据访问与清理策略 |
| `backend/app/services/snapshot.py` / `audit.py` / `snapshot_hooks.py` | 业务逻辑与自动快照钩子 |
| `backend/app/api/v1/snapshots.py` / `audit.py` | REST API |
| `frontend/admin/snapshots.html` / `audit.html` | 管理端页面 |

协作约定：

1. 其他写操作前调用 `take_auto_snapshot(...)`
2. 回退后索引为 `building`，向量化模块重建后调用 `activate_index_version`
3. `user` / `knowledge_base` / `document` 等模型为本模块依赖桩，完整实现由对应同学负责
